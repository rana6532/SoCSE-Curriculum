## 1. Course description 🟢

This course introduces students to full-stack web application development using modern open-source tools. Learners will build responsive user interfaces, design RESTful backends, integrate databases, and deploy a live web application with attention to security and performance.

---

## 2. Course content (units & weightage) 🟢

| Unit | Syllabus | Weightage |
|---|---|---|
| 1 | Web application fundamentals, HTML, CSS, responsive layout | 25 Marks |
| 2 | JavaScript, DOM programming, client-side state management | 25 Marks |
| 3 | Server-side development with Node.js and Express, REST APIs | 25 Marks |
| 4 | Database integration, authentication, deployment, performance, project | 25 Marks |

---

## 3. 🔵 Dual-level scope of each unit (KEYSTONE)

| Unit | Awareness level - every student must reach (floor) | Advanced level - required to exceed 8 CGPA (ceiling) |
|---|---|---|
| 1 | Understand browser architecture, build semantic HTML and responsive CSS, use developer tools for debugging | Implement component-based layout patterns, responsive design best practices, and accessibility improvements for production-ready pages |
| 2 | Write JavaScript programs, manipulate the DOM, validate form input, and build interactive UI behaviours | Design modular client-side state and event handling, implement asynchronous data flows, and optimise UI rendering for complex user interactions |
| 3 | Build a RESTful backend with Node.js/Express, handle requests/responses, and use middleware for routing | Design secure, scalable API contracts, implement authentication/authorization flows, and apply error handling/logging for production services |
| 4 | Integrate a database, persist user data, deploy a web application, and perform basic security checks | Build a full-stack capstone app with end-to-end functionality, apply deployment automation, performance tuning, and web security hardening |

---

## 4. Course objectives 🟢

- Introduce modern front-end and back-end web development tools and workflows.
- Enable students to develop, test, and deploy a basic full-stack web application.
- Build skills in application implementation, debugging, and quality assurance for web systems.
- Foster teamwork, documentation practices, and presentation skills through a capstone demo project.

## 8. 🔵 Differentiated instruction (mapped to the two levels)

**Reaching the floor (awareness level) - for students at risk:**
- Step-by-step guided lab sheets with sample code and checkpoints for each hands-on activity.
- Starter templates for HTML, CSS, JavaScript, and Node.js exercises to reduce syntax-related difficulties.
- Debugging checklists to help students identify and correct common HTML, CSS, JavaScript, and Node.js errors.

**Reaching the ceiling (advanced level) - for students aiming above 8 CGPA:**
- Optional advanced mini-project extensions such as role-based access, search filters, or real-time updates.
- Stretch tasks that require building custom UI components, designing API contracts, and applying caching or lazy loading.
- Self-directed exploration of deployment pipelines, containerization, or progressive web app features.

## 22. 🔵 Implementation strategy — Merrill's First Principles

> The lesson-plan table in §11 carries the actual session implementation. This section explains the strategy behind the phases and ensures the course covers them.

**Merrill's five phases:**
| Phase | What it means | Typical activity |
|---|---|---|
| Problem / Task-centred | Anchor learning in a real-world problem | Present a motivating use case or bug-hunt task |
| Activation | Activate prior knowledge | Lecture + demonstration, recap quiz |
| Demonstration | Show how to solve the problem | Live coding / worked example |
| Application | Learner practises with guidance | Guided exercise / lab |
| Integration | Learner transfers skill independently | Mini-project, peer review |

> Design rule: every unit should use all five phases, and advanced-level Application/Integration must appear in Units 2-4.

### 22.1 Phase-coverage check 🔵

| Check | Entry |
|---|---|
| All five Merrill phases used at least once per unit | Yes |
| §11 sessions using Activation = lecture + demonstration | 01, 04, 07, 10 |
| §11 sessions delivering Advanced-level activities | 06, 09, 11, 12, 13, 14 |
| ✅ Ceiling taught (≥1 Adv activity per unit, not only assessed) | Yes |
