## 1. Course description 🟢

This course introduces students to fundamental data structures and their implementations using a programming language. Students learn to choose appropriate structures for solving computational problems, analyse their efficiency, and build reliable solutions using modular programming and algorithmic reasoning.

---

## 2. Course content (units & weightage) 🟢

| Unit | Syllabus | Weightage |
|---|---|---|
| 1 | Introduction to data structures, abstract data types, arrays, strings, recursion, time and space complexity | 25 Marks |
| 2 | Linked lists, stacks, queues, applications and implementation strategies | 25 Marks |
| 3 | Trees, binary search trees, heaps, hashing, balanced trees basics | 25 Marks |
| 4 | Graphs, traversal algorithms, sorting and searching, algorithm design and case studies | 25 Marks |

---

## 3. 🔵 Dual-level scope of each unit (KEYSTONE)

| Unit | Awareness level — every student must reach (floor) | Advanced level — required to exceed 8 CGPA (ceiling) |
|---|---|---|
| 1 | Define abstract data types, implement arrays/strings and simple recursive routines, and explain basic complexity | Compare implementations, justify choice of representation, and analyse trade-offs in recursive and iterative approaches |
| 2 | Implement singly linked lists, stacks, and queues and solve standard textbook applications | Design custom linked structures, optimise operations, and handle edge cases in real-world applications |
| 3 | Construct BSTs, heaps and hash tables and perform standard insertion/deletion/search operations | Analyse balancing needs, compare tree and hashing strategies, and justify advanced structures for performance-critical cases |
| 4 | Apply graph traversal and common sorting/search algorithms to standard problems | Design efficient graph-based solutions, compare algorithmic strategies, and justify the best approach for novel problems |

---

## 4. Course objectives 🟢

- Introduce students to the concepts of data abstraction and the use of standard data structures.
- Develop problem-solving ability through implementation of algorithms on arrays, linked structures, trees, and graphs.
- Build competence in selecting appropriate data structures based on efficiency and application needs.

## 6. Pedagogy 🟢

Problem-based learning, guided programming demonstrations, flipped discussion of algorithmic trade-offs, collaborative lab practice, and mini-project-based learning.

## 7. Textbooks, references, e-resources 🟢

**Textbook(s):** Narasimha Karumanchi, Data Structures and Algorithms Made Easy; Mark Allen Weiss, Data Structures and Algorithm Analysis in C/C++.

**References:** Seymour Lipschutz, Data Structures; E. Horowitz, S. Sahni, and S. Rajasekaran, Fundamentals of Data Structures; T. H. Cormen, Introduction to Algorithms.

**Web / e-books / NPTEL:** NPTEL Data Structures and Algorithms courses, GeeksforGeeks tutorials, Visualgo for visualising data structures.

## 8. 🔵 Differentiated instruction (mapped to the two levels)

**Reaching the floor (awareness level) — for students at risk:**
- Remedial tutorials triggered by IA performance and lab checkpoints.
- Curated worked examples, pseudocode templates, and peer mentoring using short practice sheets.
- Scaffolded programming tasks that build confidence in implementation basics.

**Reaching the ceiling (advanced level) — for students aiming above 8 CGPA:**
- Advanced problem sets involving edge cases, optimisation, and comparison of approaches.
- Short concept notes or paper reviews on efficient data-structure choices in real systems.
- Mini-projects requiring justification of data-structure choices for novel application scenarios.
